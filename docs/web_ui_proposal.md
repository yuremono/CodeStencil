# CodeStencil Web UI 提案

> **作成日**: 2026-02-07
> **担当**: @member3
> **視点**: 差別化・市場性

---

## 提案タイトル: コンテキスト Aware なテンプレート生成 UI

---

## 概要

CodeStencilは「静的なスニペット集」ではなく、「プロジェクトのコンテキストを理解して動的にテンプレートを生成する」ことが最大の特徴です。Web UIではこの差別化ポイントを前面に押し出し、既存サービス（V0.dev, StackBlitz）とは明確に違う価値を提供します。

---

## 既存サービスとの比較

| サービス | 主な機能 | CodeStencilの差別化 |
|---------|---------|---------------------|
| **V0.dev** | AIによるUI生成 | コードスニペットの生成に特化 |
| **StackBlitz** | オンラインIDE | プロジェクト解析とスタイル学習 |
| **CodeSandbox** | コードエディタ | RAGによる類似パターン検索 |
| **GitHub Copilot** | コード補完 | テンプレートとしての再利用性 |

---

## 提案するUIコンセプト

### 1. 「プロジェクト接続」が最初の体験

**既存サービスの課題**: どのサービスも、まずテンプレートを選ぶよう求めてくる

**CodeStencilのアプローチ**:
1. 最初に「プロジェクトを接続」または「コードをペースト」
2. システムがプロジェクトを解析（AST解析で使用言語、スタイルを特定）
3. 解析結果に基づいて、最適なテンプレートを提案

**UIフロー**:
```
トップページ
  ↓
「プロジェクトを接続」ボタン
  ↓
GitHub連携 または  コードペースト
  ↓
解析中...（使用言語、フレームワーク、コーディングスタイルを表示）
  ↓
プロジェクトに最適化されたテンプレートが表示
```

---

### 2. チャットベースのテンプレート生成

**参考**: V0.devのチャットUI

**実装**:
- 左側: チャット画面（自然言語で要件を入力）
- 右側: 生成されたテンプレートのプレビュー
- プロジェクトのスタイルが自動適用されることを明示

**差別化ポイント**:
- 「あなたのプロジェクトのスタイルを学習しました」バッジを表示
- 生成コードに「元のコードとの類似度」を表示
- ワンクリックでプロジェクトに追加

---

### 3. テンプレートカード UI

**参考**: Notionのテンプレートギャラリー

**カードに表示する情報**:
| 項目 | 説明 |
|------|------|
| サムネイル | コードのプレビュー（シンタックスハイライト） |
| タイトル | テンプレート名 |
| タグ | 使用言語、フレームワーク |
| 類似度 | プロジェクトとの類似度（RAGスコア） |
| 使用回数 | チーム内での使用回数 |

## UIデザインの方向性

### シンプル vs 機能豊富

**結論**: シンプルに、だがプロジェクト解析は「目立つように」

**理由**:
- ユーザーは「テンプレートを探す」のではなく、「自分のプロジェクトに最適なものを得たい」
- そのため、プロジェクト解析結果が最初に目に入るべき
- テンプレート一覧はシンプルに（カード形式）

## 既存サービスから参考にすべき点

| サービス | 参考点 | 導入方法 |
|---------|--------|----------|
| **V0.dev** | チャットUIの自然さ | 左側チャット、右側プレビュー |
| **StackBlitz** | コード実行の即時性 | WebContainerでのライブプレビュー |
| **Notion** | テンプレートギャラリー | カード形式、タグフィルター |
| **GitHub** | プロジェクト連携の簡単さ | OAuthでワンクリック接続 |

## 差別化の要点

1. **「プロジェクトを理解する」ことを前面に**
   - 「あなたのプロジェクトを解析しました」を最初に表示
   - 発見したスタイル（命名規則、インデント等）を表示

2. **RAGによる「類似パターン」の提示**
   - 「あなたのプロジェクトの類似コード」を表示
   - 生成コードがプロジェクトに馴染むことを実感させる

3. **「ワンクリックで追加」体験**
   - GitHub連携で直接プロジェクトに追加
   - ファイルパスを指定して配置

---

## まとめ

CodeStencilのWeb UIは、「テンプレート集サイト」ではなく、「プロジェクトの相棒」としての位置づけが必要です。既存サービスは「テンプレートを探す」体验ですが、CodeStencilは「プロジェクトを解析して、最適なものを提案する」体験を提供します。

これが差別化の核心です。
